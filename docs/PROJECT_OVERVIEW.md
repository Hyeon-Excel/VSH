# VSH (Vibe Secure Helper) — 프로젝트 기획의도 및 개요

## 1. 기획 의도 (Why)

최근 Copilot, Claude Code, Gemini 등 코딩 AI의 보급으로 “바이브 코딩(vibe coding)”이 보편화되면서 **코드 생산 속도는 급격히 증가**했지만, 그 속도를 따라가지 못하는 **보안 검증 공백**이 동시에 커지고 있다.  
특히 AI가 생성한 코드는 개발자가 내부 로직을 충분히 이해·검증하지 못한 채 적용되는 경우가 많아, 기존의 SQL Injection/XSS 같은 전통적 취약점부터 **권한 체크 누락, 민감정보 노출 등 비즈니스 로직 취약점**, 그리고 **환각 패키지/타이포스쿼팅 기반 공급망 공격**까지 위험이 확대된다.

이에 따라 본 프로젝트는 **“AI가 코드를 작성하는 순간(IDE 작업 흐름)에서 보안을 끼워 넣는 도구”**를 목표로 한다.  
단순히 “위험하다”를 경고하는 수준이 아니라, **국내 기준(KISA 시큐어코딩, 금융보안원 체크리스트 등)과 국제 기준(OWASP Top 10, CWE)을 근거로** 탐지 사유를 설명하고, **수정 Diff 제안 및 최종 리포트**를 생성해 개발자와 조직이 동시에 활용할 수 있도록 한다.

---

## 2. 프로젝트 개요 (What)

### 2.1 한 줄 소개

**VSH는 AI-agnostic(코딩 AI와 무관)하게 동작하는 IDE 내 보안 비서로, 코드 생성 직후/저장 시점에 취약점·공급망 리스크를 탐지하고 근거 기반 수정안을 제시하는 로컬 실행형 보안 검증 도구이다.**

### 2.2 해결하려는 핵심 문제

- **보안 부채(Security Debt) 증가:** AI 생성 코드가 빠르게 누적되며, 리뷰/검증이 따라가지 못함
- **공급망 공격 위험:** 환각 패키지, 타이포스쿼팅 등으로 악성 의존성이 유입될 가능성
- **근거/리포트 부재:** “왜 위험한지”와 “어떤 기준을 만족하는지”를 증빙하기 어려움
- **검증 타이밍 문제:** PR/배포 단계가 아니라 **코딩 순간에 개입**해야 비용 대비 효과가 큼

### 2.3 목표(Goals)

- IDE 흐름을 방해하지 않는 속도로 **즉시(L1) 경고**를 제공한다.
- 취약점 탐지 결과를 **근거(가이드라인/표준)와 함께 설명**한다.
- 수정안은 **Diff 형태로 제시**하며, 반영은 개발자가 승인(Human-in-the-Loop)한다.
- 프로젝트 완료 시점에 **최종 보안 진단 리포트(MD/JSON)**를 생성해 제출/감사에 활용 가능하게 한다.

### 2.4 비목표(Non-goals)

- “완전 자동 수정 후 배포”처럼 최종 의사결정을 AI가 대신하지 않는다.
- 모든 언어/모든 취약점의 완전 탐지(100%)를 목표로 하지 않는다.
- 고급 정적 분석(전 언어 크로스파일 정밀 데이터플로우 등)은 MVP 범위에서 제외하고 로드맵으로 둔다.

## 3. 사용자 시나리오 (How it’s used)

### 3.1 개발 중 즉시 점검 (IDE 내)

1. 사용자가 Copilot/Claude/Gemini 등으로 코드 생성 또는 수정
2. 사용자가 `VSH Scan`을 실행하거나 저장 시 자동으로 빠른 스캔 수행
3. 취약점/위험 요소를 IDE 알림(진단/주석)으로 표시
4. 필요 시 “수정안 생성”을 눌러 Diff 미리보기 → 개발자 승인 후 적용

### 3.2 제출/배포 전 최종 리포트 생성

- 전체 프로젝트를 대상으로 심층 스캔을 수행하고,
- 취약점 요약, 기준 준수율(KISA/OWASP 등), SBOM 요약, 개발자 조치 로그(HITL)를 포함한
  **최종 보안 진단 리포트**를 출력한다.

---

## 4. 핵심 특징 (Key Features)

- **AI-agnostic:** 특정 코딩 AI에 종속되지 않고 IDE/프로젝트 기준으로 동작
- **3-Layer 분석:**
  - L1 Hot: 빠른 정적 분석 기반 즉시 경고
  - L2 Warm: 근거(RAG) + 수정 Diff 제안
  - L3 Cold: 전체 프로젝트 스캔 + SBOM 기반 최종 리포트
- **Explainable Security:** 결과에 “근거(가이드/표준)”를 포함해 설명 가능
- **Human-in-the-Loop:** 최종 수정 반영은 개발자 승인으로 강제

---

## 5. 기대 효과 (Impact)

- 개발 단계에서 취약점을 조기에 발견하여 **수정 비용과 리스크를 절감**
- 환각 패키지/타이포스쿼팅 등 공급망 공격을 **유입 시점에서 차단**
- 근거 기반 리포트로 조직의 **컴플라이언스/감사 대응 역량 강화**
- “AI가 만든 코드”의 신뢰도를 높여 **AI 활용 속도와 보안 수준을 함께 끌어올림**
